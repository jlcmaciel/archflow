The problem

Architects are stuck in
desktop software from 2005.

Four tools dominate a billion-dollar market. All of them were designed for a single user on a single machine. None of them match how creative teams actually work in 2026.

SketchUp

Concept-phase king, but stagnant for a decade. File-sync "collaboration". BugSplats on large models. No path to BIM.

Revit · $335/mo

Industry BIM standard. Brutal UX. Useless for early concept design — everyone models twice.

Friendlier than Revit. Desktop-only. File incompatibilities between versions. Expensive for solo architects.

AutoCAD · $2k/yr

The 2D lingua franca. Legacy workflow. Client presentations still happen through PDF screenshots.

FAQ

Common questions.

Is this a replacement for Revit?

Not in v1. ArchFlow focuses on conceptual and schematic design — the first 2–8 weeks of a project. We export to IFC so your concept feeds into Revit or ArchiCAD for construction documents. The promise: stop modeling twice.

Does it work offline?

The editor runs in the browser with local-first persistence. You can edit offline; changes sync when you reconnect. Real-time collaboration naturally requires a connection.

Who owns the model?

You do. Every project exports to open formats: IFC, GLB, OBJ, DWG. We'll never lock you in.

What about rendering?

We integrate with Enscape, Lumion, D5, and Twinmotion via GLB export. ArchFlow is the model, not the render.

Which browsers are supported?

Chrome, Edge, Firefox, and Safari (latest two versions). Performance is best on Chromium with WebGPU enabled.

Is my data secure?

Projects are encrypted at rest and in transit. SSO and role-based access control ship in the Studio plan. We're SOC 2 Type I in progress.
